# ADR-000: huginn — Project Foundation

- **Status**: Accepted
- **Date**: Generated by HAMR CLI

## Context

This project was scaffolded using the HAMR framework (`github.com/FyrmForge/hamr`).

## Stack

- **Language**: Go 1.26.4
- **HTTP**: Echo v4 (via `hamr/pkg/server`)
- **Templates**: Templ + HTMX
- **Database**: PostgreSQL (via `hamr/pkg/db`)
- **CSS**: tailwind
- **Sessions**: Cookie-based (via `hamr/pkg/auth`)
- **Auth**: Argon2id password hashing + session management
- **Storage**: Local filesystem (via `hamr/pkg/storage`)
- **WebSocket**: Session-based hub (via `hamr/pkg/websocket`)
- **E2E Testing**: Go-rod + Testcontainers

## Project Structure

```
cmd/site/          Entry point + Dockerfile (env config loaded here)
internal/db/         Database connection + migrations
internal/repo/       Data access layer (Store interface)
internal/web/        HTTP handlers + components
static/              CSS, JS, images
docker/              Docker Compose
docs/                Documentation
```

## Conventions

- Handlers use constructor injection and register via `web.RegisterRoutes`
- Route comments: `// GET /path` above handler methods
- Validation: `hamr/pkg/validate` validators return "" for valid, error message for invalid
- Response: `hamr/pkg/respond.HTML` for templ components, `respond.JSON` for API responses
- Config: env vars read in `main.go` via `hamr/pkg/config` helpers
